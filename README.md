# E-Commerce-Manager
企业级电商业务后台管理系统

## 项目简介
E-Commerce-Manager是一个基于Vue 3和TypeScript开发的企业级电商后台管理系统，旨在为电商行业提供全流程的后台管理能力，通过直观的界面和高效的操作提升管理效率。

## 核心功能特性

### 商品管理
- **商品列表**：支持商品多维度筛选、排序、搜索，展示商品基本信息与库存状态
- **添加商品**：完整的商品信息录入，包括基本信息、SKU配置、图片上传、属性设置
- **分类管理**：多级商品分类体系，支持分类的增删改查与排序
- **规格管理**：商品规格参数的定义与管理，支持规格组合配置
- **库存管理**：实时库存监控与调整，支持库存预警与批量操作

### 订单管理
- **订单列表**：全量订单数据展示，支持按订单状态、时间、金额等多维度筛选
- **订单详情**：完整的订单信息展示，包括商品明细、收货信息、支付状态、物流信息
- **退款管理**：退款申请处理、审核流程、退款状态跟踪与管理

### 用户管理
- **用户列表**：系统用户管理，支持用户信息查看、编辑、禁用/启用
- **权限控制**：基于角色的权限管理，细粒度控制菜单、按钮、数据访问权限
- **角色管理**：自定义角色创建与管理，灵活分配权限组合
- **菜单管理**：系统菜单配置，支持菜单的增删改查与排序

### 商家管理
- **商家列表**：商家信息管理，支持商家搜索、筛选、状态管理
- **商家审核**：新商家入驻审核流程，包括资质审核、信息验证、审核状态跟踪

### 财务管理
- **资金管理**：商家资金账户管理，包括资金流水、提现申请处理
- **发票管理**：发票申请、开具、管理与查询
- **报表统计**：多维度财务数据统计与分析，支持图表展示与数据导出

### 客户服务
- **投诉管理**：客户投诉的接收、处理、跟踪与归档
- **工单管理**：客户服务工单的创建、分配、处理与状态更新

### 系统管理
- **操作日志**：系统所有操作的详细记录，支持按用户、操作类型、时间等筛选
- **个人中心**：用户个人信息管理、密码修改、操作历史查看

## 技术栈说明
### 核心框架
- Vue 3：前端框架
- TypeScript：类型系统
- Vite：构建工具
- SCSS：样式预处理器

### 依赖库
- Element Plus：UI组件库
- Pinia：状态管理
- Vue Router：路由管理
- Axios：网络请求

### 开发工具
- TypeScript：类型检查
- Sass：样式编译
- Vite：开发服务器与构建

## 环境要求
- Node.js：v16.0.0+（推荐v18.0.0+）
- 包管理器：npm 7.0.0+ 或 yarn 1.22.0+

## 快速开始
### 1. 克隆项目
```bash
git clone https://github.com/youkl1/E-Commerce-Manager.git
cd E-Commerce-Manager
```

### 2. 安装依赖
```bash
npm install
# 或
yarn install
```

### 3. 本地启动
```bash
npm run dev
# 或
yarn dev
```

### 4. 生产构建
```bash
npm run build
# 或
yarn build
```

## 贡献指南
1. **分支规范**：
   - `main`：主分支，用于生产环境
   - `develop`：开发分支，用于集成测试
   - `feature/xxx`：功能分支，用于开发新功能
   - `fix/xxx`：修复分支，用于修复bug

2. **提交规范**：
   - 格式：`type(scope): subject`
   - 示例：`feat(product): 添加商品分类功能`
   - 类型：feat（新功能）、fix（修复）、docs（文档）、style（样式）、refactor（重构）、test（测试）、chore（构建）

3. **PR提交流程**：
   - 从 `develop` 分支创建功能分支
   - 完成代码开发和测试
   - 提交PR到 `develop` 分支
   - 代码审查通过后合并

### 常见问题FAQ
- **Q**: 如何配置环境变量？
  **A**: 复制 `.env.example` 文件为 `.env`，并根据实际情况修改配置。

- **Q**: 本地开发时遇到端口被占用怎么办？
  **A**: 在 `vite.config.ts` 中修改 `server.port` 配置，或使用 `npm run dev -- --port 3000` 指定端口。
