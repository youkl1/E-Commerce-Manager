# 多角色多平台电商管理系统需求对齐文档

## 一、项目与任务特性规范

### 1.1 项目基本信息
- **项目名称**：多角色多平台电商管理系统
- **项目类型**：企业级电商后台管理系统
- **技术栈**：Vue 3 + TypeScript + Vite + Element Plus + Pinia + Vue Router + Axios
- **目标用户**：平台管理员、平台运营、商家管理员、财务人员、客服人员

### 1.2 任务边界确认
- **做什么**：
  - 实现基于 RBAC 模型的前端权限管控
  - 实现多平台适配（自营平台、第三方平台）
  - 实现核心业务模块：系统管理、商品管理、订单管理、商家管理、数据统计
  - 实现响应式布局，适配 PC 端管理后台

- **不做什么**：
  - 不实现后端逻辑
  - 不实现数据库设计
  - 不实现服务端部署配置
  - 不实现 Docker 相关内容

## 二、原始需求原文

### 2.1 核心业务定义
1.  **多角色体系**：实现基于 RBAC 模型的前端权限管控，不同角色登录后仅可见对应权限的菜单、页面与操作按钮，角色定义如下：
    - 超级管理员：平台全量权限，含租户/商家管理、平台配置、角色权限管理、全平台数据看板
    - 平台运营：商品审核、商家入驻审核、营销活动管理、全平台订单监管、内容管理
    - 商家管理员：所属店铺管理、商品上下架、订单处理、库存管理、会员管理、财务对账
    - 财务角色：订单对账、资金流水管理、发票管理、退款审核、财务报表导出
    - 客服角色：售后工单处理、客诉管理、订单咨询跟进、退款流程处理

2.  **多平台适配**：
    - 多电商平台统一管理：支持自营平台、淘宝、京东、拼多多等第三方平台的订单、商品、库存数据统一管理视图
    - 多租户前端适配：不同商家/租户登录后，实现前端数据隔离与视图隔离，仅展示所属租户的相关数据
    - 响应式布局：适配 PC 端管理后台主流显示器尺寸，兼容 1366px-1920px 宽度区间

### 2.2 技术架构与开发规范
1.  **工程目录结构**：严格按照规范的 Vue 项目目录结构组织代码
2.  **组件设计**：容器/展示分离，容器组件管理数据与业务逻辑，展示组件仅通过 props 接收数据渲染 UI
3.  **路由规范**：实现基于角色的动态路由加载，登录后根据用户角色动态挂载对应权限的路由，配合全局路由守卫实现鉴权与重定向
4.  **状态管理**：Pinia 按业务模块拆分，必须包含 user 用户状态、permission 权限状态、app 全局布局状态三大核心 store，其他业务模块按需拆分
5.  **请求封装**：Axios 统一封装请求/响应拦截器，处理 Token 鉴权、接口报错统一提示、请求取消、重复请求拦截
6.  **权限控制**：实现菜单权限、按钮权限两级管控，封装全局权限指令 v-hasPerm，控制按钮级别的操作权限
7.  **样式规范**：使用 SCSS 预处理器，组件样式使用 scoped 隔离，避免全局样式污染，支持 Element Plus 主题定制
8.  **代码注释**：关键逻辑、组件功能、类型定义、复杂方法必须添加清晰的中文注释，便于维护

### 2.3 核心功能模块（按优先级排序）
1.  **系统基础模块**：登录页、全局布局框架、角色权限管理、用户管理、菜单管理、个人中心、操作日志
2.  **商品管理模块**：商品列表、商品新增/编辑、分类管理、规格管理、库存管理、商品审核、上下架管理
3.  **订单管理模块**：全平台订单列表、订单详情、订单状态流转、发货管理、退款售后管理、订单筛选与导出
4.  **商家/租户管理模块**：商家入驻审核、商家信息管理、商家权限配置、店铺状态管理
5.  **数据统计模块**：平台销售数据看板、商家经营数据、商品转化数据、订单数据统计图表

## 三、需求理解说明

### 3.1 多角色权限管控
- 基于 RBAC（基于角色的访问控制）模型，实现前端权限管理
- 不同角色拥有不同的菜单权限和操作权限
- 登录后根据用户角色动态加载对应权限的路由
- 实现按钮级别的权限控制，通过全局指令 v-hasPerm 实现

### 3.2 多平台适配
- 支持多电商平台数据统一管理，包括自营平台和第三方平台（淘宝、京东、拼多多等）
- 实现多租户数据隔离，不同商家登录后只能看到自己店铺的数据
- 响应式布局，适配 PC 端不同尺寸的显示器

### 3.3 核心业务模块
- **系统基础模块**：提供系统管理的基础功能，包括用户管理、角色管理、菜单管理等
- **商品管理模块**：管理商品的全生命周期，从创建、编辑到上下架、库存管理
- **订单管理模块**：处理订单的流转、发货、退款等操作
- **商家管理模块**：管理商家的入驻、信息、权限等
- **数据统计模块**：提供数据看板，展示平台和商家的经营数据

## 四、现有项目适配分析

### 4.1 技术栈适配
- 项目使用 Vue 3 + TypeScript + Vite 作为基础技术栈，符合要求
- 使用 Element Plus 作为 UI 组件库，满足管理后台的需求
- 使用 Pinia 进行状态管理，符合要求
- 使用 Vue Router 4 进行路由管理，符合要求
- 使用 Axios 进行网络请求，符合要求

### 4.2 架构适配
- 采用模块化、组件化的架构设计，符合要求
- 实现了基于角色的动态路由加载，符合要求
- 实现了权限指令，控制按钮级别的权限，符合要求
- 采用 SCSS 预处理器，支持主题定制，符合要求

## 五、歧义与疑问澄清清单

| 序号 | 歧义点 | 描述 | 决策依据 | 结论 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | 多平台数据同步机制 | 如何实现多平台数据的同步和统一管理 | 基于前端视角，仅实现数据展示和管理界面，具体同步机制由后端实现 | 前端仅提供统一的管理界面，数据同步由后端处理 |
| 2 | 多租户数据隔离实现方式 | 如何在前端实现多租户数据隔离 | 基于用户角色和租户ID，在前端请求时携带租户信息，后端根据租户ID返回对应数据 | 前端通过状态管理存储租户信息，请求时携带租户ID |
| 3 | 数据统计图表库选择 | 未指定数据统计图表库 | 考虑到 Element Plus 生态，选择 ECharts 作为图表库 | 使用 ECharts 实现数据统计图表 |
| 4 | 第三方平台 API 集成 | 如何集成第三方平台的 API | 前端仅提供集成界面，具体 API 调用由后端实现 | 前端提供集成配置界面，后端处理 API 调用 |
